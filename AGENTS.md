# AGENTS.md — инструкции для ИИ-ассистентов

## Правила работы

- Если задачу можно решить без кода — расскажи как, не пиши код
- Если пользователь предлагает сложную правку — попробуй упростить, если это возможно
- Если пользователь что-то упускает или не прав — укажи на проблему и предложи решение
- KISS и DRY — главные принципы
- Основная цель — краткий и максимально читаемый код с хорошим форматированием
- Старайся не просто показать решение, а объяснить, почему именно это решение выбрано и в чём его преимущество перед альтернативами
- После завершения фичи (фразы «готово», «заработало», «всё», «молодец») — спросить, не пора ли обновить SUMMARY.md

## Особенности проекта

- **Русский язык во всём проекте.** Пользователь не знает английского.
  Весь UI, комментарии, логи, сообщения — на русском.
  Исключение: общепринятые названия (JUCE, VST3, MIDI, JSON, plugin, host),
  названия файлов, технические идентификаторы (классы, функции, переменные).

## Нейминг

### Переменные, классы, функции

- Имя должно отражать **назначение**, а не значение:
  - ✅ `buttonStart`, `textPrimary`, `sliderTrack`
  - ❌ `buttonBlue`, `textDark`, `sliderGrey`
- Каждая переменная хранит собственное значение, а не ссылку на другую:
  - ✅ `editorBg = Colour(0xFF13294B)`, `editorText = Colour(0xFFE0EDFF)`
  - ❌ `editorBg = bgPanel`, `editorText = textPrimary`
- Взаимные ссылки — только по явному запросу разработчика

### Файлы

- Формат: `PascalCase` (C++-стиль):
  - ✅ `PluginEditor.cpp`, `ControlsPanel.h`, `Song.cpp`
  - ❌ `plugin_editor.cpp`, `controls_panel.h`
- Имя отражает суть содержимого, контекст + специализация:
  - ✅ `TabStatus.h`, `TabSettings.h`, `TabLogs.h`
  - ❌ `StatusTab.h`, `SettingsTab.h`, `LogsTab.h`
- **Запрещены файлы-двойники** с одинаковым именем во всём проекте (без учёта пути):
  - ❌ `Source/UI/audio.h` + `Source/Data/audio.h`
  - Если имя занято — отразить различие контекстов в имени

### Пути

- Не хардкодить пути. Только `%USERPROFILE%`, `%LOCALAPPDATA%`, `%APPDATA%` или API фреймворка (загрузи `cpp-single-source`)

### Списки

- Если элементов в списке больше 3 — предлагать разработчику вынести в отдельный JSON
