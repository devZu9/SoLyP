# Roadmap / Планы по рефакторингу

## Цель

Переписать отрисовку текста с «координатной» модели на простую
«print(строка_в_окне, текст) + scroll(speed)».

## Нынешняя модель (проблемная)

- `scrollHead` — абстрактный сдвиг, не равный индексу строки
- Реальный индекс строки = `scrollHead + preLinesOnPause - visibleLines + номер_ячейки`
- Из-за этого `scrollHead = 4` означает «первая строка видна вверху»,
  а `scrollHead = 0` — «ничего не видно»
- Всё перерисовывается 20 раз в секунду, позиция каждой строки
  пересчитывается с нуля

## Желаемая модель (простая)

Размер окна выстраивается пользователем и им же настраивается количество строк. Подгоняется всё друг под друга автоматически, исходя их размеров шрифта (уже реализовано). Для примера возьмём что окно имеет 6 фиксированных строк, где каждая строка имеет номер и индекс начинается с нуля. Т.е: 

Окно — 6 фиксированных ячеек с номерами 0..5.
Номер ячейки соответствует вертикальной позиции на экране сверху вниз.

### Функции

```
cls()                           // очистить экран
print(номер_ячейки, текст)      // напечатать строку в ячейке
scroll(скорость)                // сместить всё на 1 ячейку вверх за время,
                                // рассчитанное по BPM и следующей строке
```

### Жизненный цикл

#### Старт (загрузили песню, STOP)

Знаем: первая строка секции = `displayLines[0]`.
Окно 6 строк (рассчитывается): visibleLines = 6

Предварительно показанные строки (берётся из настроек), preLines = 2.

Индекс для следующей играющей строки сейчас в стартовой позиции: nextLineIndex = 0

```
// подготовка
cls()    // очищает весь экран, нужно, если необходимо очистить всё в начале песни или при смене секции, 
         // в других местах в cls пока нет необходимости
preLineIndex = 0  // только чтобы рисовать предстроки в последовательности сверху вниз
specForCycle = nextLineIndex
for i = specForCycle ..specForCycle + (preLines - 1):
    print(visibleLines - preLines + nextLineIndex, displayLines[i])
    preLineIndex++ 
    nextLineIndex++
// print(4, displayLines[0])   // 1-я предстрока (предстрока это не "предыдущая", а "предварительно показанная")
// print(5, displayLines[1])   // 2-я предстрока
// строки (ячейки) 0..3 пустые
// таймер НЕ идёт
// playLineIndex = 2 (т.е. следующая строка будет третья)
```

#### Play (после предстрок)

```
// т.к. предстроки уже на экране, ничего очищать не нужно, мы просто их будем смещать 

specForCycle = nextLineIndex
for i = specForCycle ..len(displayLines)-1:
    scroll(скорость = 60/BPM * timeSig / linesPerBar / parts (найдено используя playLineIndex))
    // за это время всё окно смещается на 1 строку вверх
    
    print(5, displayLines[i])
    nextLineIndex++

// displayLines[0] становиться выше на одну строку (т.е. ячейка 3)
// displayLines[1] переходит на позицию 4
// print(5, displayLines[2]) // displayLines[2] снова появляется на позиции 5 (нижняя строка) и так все последующие


```

На каждом тике таймера:

```
cls()
for i = 0..visibleLines-1:
    строкаИндекс = nextLineIndex + i - (visibleLines - preLines)
    if строкаИндекс >= 0:
        print(i, displayLines[строкаИндекс])
```

Где `nextLineIndex` — индекс следующей новой строки (которая появится снизу).

### Планируемые изменения

1. Заменить `scrollHead` на `nextLineIndex` — индекс строки, которая
   появится снизу следующей
2. Отрисовка: Цикл вызывающий смещение при помощи scroll и всегда рисующий очередную строку (с прямым индексом) в нижней позиции окна. Иногда при необходимости очищается окно при помощи cls()
3. `scroll(speed)` — функция, которая смещает весь экран окна за указанное время на 1 строку, всё что ушло за верхние пределы окна забывается
4. Пауза: таймер останавливается, экран застывает, предстроки рисуются
   в нижних ячейках (как описано в коде выше)
5. После паузы: таймер запускается с той же `nextLineIndex`, скролл
   продолжается
6. Stop: таймер останавливается, экран НЕ меняется

### Преимущества

- Никаких `scrollHead`, `pauseLineDisplayIdx`, `resumeSkipIdx`
- Никаких формул с дробными сдвигами
- Индекс строки = реальный индекс в `displayLines`
- Код отрисовки — 10 строк вместо 100+
- Баги с прыжками текста исчезают сами собой
