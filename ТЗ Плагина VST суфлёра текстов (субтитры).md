# Техническое задание: VST3/AU/Standalone плагин «Суфлёр текста»

Английский вариант названия: SoLyP (song lyrics prompter)

## 1. Общее описание

Плагин отображает крупный текст песни (суфлёр) для уличных музыкантов.  
Работает синхронно с тактовой сеткой DAW (или MIDI Clock в Standalone).  
Управление через MIDI-ноты двух типов:

- **Ориентиры** (октава 1 по умолчанию) – переключают активную секцию текста.
- **Триггеры** (октава 2 по умолчанию) – команды «плей», «пауза», «следующая секция», «гибрид» и др.

Плагин не использует абсолютное время – только целые такты (каждый такт → следующая строка внутри секции).  
Формат данных песни – JSON с легендой привязки нот к секциям.

## 2. Платформы и форматы

- Windows, macOS (Apple Silicon + Intel)
- VST3, AudioUnit (macOS), Standalone приложение
- Рекомендуемый фреймворк: **JUCE** (7.x) – даёт кроссплатформенность, MIDI/аудио обработку, UI.

## 3. Основной функционал

### 3.1 Отображение текста

- Окно плагина: настраиваемое количество строк (ползунок, появляется при наведении мыши в углу окна).
- Настраиваемый размер шрифта (ползунок, там же).
- Выбор шрифта (параметр в отдельном меню общих настроек).
- Поведение при слишком длинной строке (параметр в общих настройках):  
    – перенос на следующую строку,  
    – автоматическое уменьшение шрифта только для этой строки.
- Полная поддержка UTF-8 без BOM (кириллица, эмодзи, диакритика).
- Режим «пауза» – показывает текст «(ждём, не поём)» (настраивается в общих настройках).

### 3.2 MIDI-управление

- Плагин слушает MIDI-вход (канал можно выбрать в настройках, по умолчанию all).
- **Октава ориентиров** (по умолчанию: C1–B1). Каждая нота соответствует секции через легенду в JSON.
- **Октава триггеров** (по умолчанию: C2–B2).
- Назначение триггеров по умолчанию (настраивается в настройках плагина):
    - C2 → Play (снять паузу, начать/продолжить показ строк по тактам)
    - D2 → Пауза (остановить смену строк, заморозить текущий текст)
    - E2 → Переход к следующей секции (без снятия/установки паузы)
    - F2 → Гибрид (переключить секцию на следующую и одновременно Play)
    - G2 → Запуск обратного отсчёта 3..2..1 (после окончания – автоматически Play)
    - A2 → Запуск 5..4..3..2..1 (опционально, может быть отключён или переопределён)
        
- Все MIDI-назначения (октавы, номера триггеров) переопределяемы в UI и сохраняются в отдельный файл настроек (JSON).
### 3.3 Синхронизация и логика работы

- Плагин получает от хоста (DAW) текущую позицию в тактах и BPM через `AudioPlayHead`.
- Переключение на следующую строку происходит **строго по началу каждого нового целого такта** (когда номер такта увеличивается).
- Если включён режим Play – каждый такт показывает следующую строку из текущей секции.
- Если секция закончилась (все строки показаны), плагин автоматически переходит к следующей секции (по порядку из JSON файла лирики) и продолжает показ, если Play активен.
    
- **Пауза**:
    - При получении команды «пауза» (по умолчанию настроена нота D2) – останавливает смену строк по тактам, но текущая строка остаётся на экране.
    - Показ дополнительной надписи (например, «(ждём, не поём)») включается сразу после паузы.
    - Снятие паузы (Play) – текст продолжается со следующей строки, начиная со следующего целого такта.
- **Ориентиры** (ноты октавы 1) переключают активную секцию, не меняя режим пауза/play. При этом на экране отображаются **предстроки** новой секции (количество настраивается в общих настройках, по умолчанию 1). _Если во время воспроизведения (Play активен) приходит ориентир или триггер «следующая секция» (E2) или переход на указанную секцию (C1,D1,E1 и т.п.), то текущая строка дотягивается до конца текущего такта, а начиная со следующего такта начинается показ строк новой секции с её первой строки. Предстроки при этом не показываются (они только для паузы)_
- **Триггер «переход к следующей секции» (E2)** работает аналогично ориентиру: переключает на следующую секцию по порядку, не меняя режим пауза/play.
- **Гибрид (F2)** – переключает секцию (на следующую) и снимает паузу (запускает Play) – начинает показ с первой строки новой секции со следующего такта.
- **Обратный отсчёт (G2, A2)** – запускает таймер обратного отсчёта (в тактах), на экране показываются крупные цифры, а под ними – первая строка целевой секции. Смена каждой цифры происходит каждые 1/2 такта. По достижении 1 – автоматически снимается пауза, и с началом следующего такта начинается показ строк (т.е. переход в режим Play). Важно понимать, что эти 3 счёта должны быть полностью показаны в своё время. Пользователь должен понимать, что включение для включения этой возможности, триггерную ноту нужно поставить (или нажать) за 1,5 такта до последующей секции с текстом (для счёта из 3) или за 2,5 такта (для счёта из 5). Почему так? Потому что цифра 3 будет показана на отметке за 1,5 такта, цифра 2, за 1 такт, цифра 1 за а 0.5 такта, а уже четвёртая доля, попадая на начало следующего такта, начинает основной Play нашего текста.

### 3.4 Формат JSON (песня + легенда)

Пример структуры (вариант, следующий из обсуждения):
```json
{
  "song": [
    {
      "section": "куплет 1",
      "lines": [
        "С гpанатою в каpмане",
        "с чекою в pуке",
        "Мне чайки здесь запели",
        "на знакомом языке"
      ]
    },
    {
      "section": "припев 1",
      "lines": [
        "Уходим уходим",
        "ухо-о-о-дим",
        "Hаступят вpемена",
        "почи-ще",
        "Бьется родная,",
        "в экстазе пылая",
        "Владивосток",
        "две тыщи"
      ]
    }
  ],
  "legends": [
    { "section": "куплет 1", "note": "C1" },
    { "section": "припев 1", "note": "C#1" }
  ],
  "settings": {
    "preLinesOnPause": 1,
  }
}
```

- `song` – массив секций (порядок определяет последовательность по умолчанию). У секции есть имя и массив строк (каждая строка соответствует одному такту).
- `legends` – привязка MIDI-нот (буквенное обозначение, система октав определяется в глобальных настройках плагина) к именам секций. Если легенда неполная, недостающие секции получают следующие ноты по порядку, начиная с заданной в настройках базовой ноты.
- `settings` – (может отсутствовать по умолчанию) переопределение поведения для конкретной песни (количество предстрок).

### 3.5 Редактор текста внутри плагина

- Пользователь может открыть окно редактирования.
- Вводит текст в формате:
```
[куплет 1]
строка 1
строка 2

[припев]
строка a
строка b
```
- Плагин автоматически парсит, создаёт внутреннюю структуру, генерирует легенду (последовательные ноты для секций берёт из настроек). Пустые строки не учитывает, они нужны только для визуального восприятия пользователя и в json не попадают.
- Возможность сохранить результат в JSON (в папку, выбранную пользователем) и загрузить позже.
- Иметь возможность мини-эксплорера с отображением списка песен (сохранённых JSON) (хранятся в Documents/SoLyP/songs/ или аналогично)
- Поддержка UTF-8 без BOM.
### 3.6 Настройки плагина (глобальные)

- Выбор системы нумерации октав (например, Yamaha, Roland, Middle C = C3/C4).
- Выбор Октавы для ориентиров (по умолчанию 1) и выбор октавы для триггеров (по умолчанию 2).
- Выбор и переназначение конкретных нот триггеров в рамках выбранной октавы, т.е. кому-то удобны только белые клавиши (целые ноты): C, D, E, F, G, A, а кто-то может использовать и любые доступные в рамках октавы C, C#, D, D# и т.д.
- Количество предстрок, показываемых при переключении секции на паузе (1–5).
- Возможность смены текста надписи при паузе (по умолчанию «(ждём, не поём)», по умолчанию загружать из нужной локали, но если уже текст был сменён пользователем, то из json локали не брать. Приоритет пользовательский).
- Поведение длинных строк: перенос или уменьшение шрифта.
- Выбор языка интерфейса (русский/английский; языковые файлы – JSON в UTF-8 без BOM). При добавлении нового файла JSON с локалями других языков, плагин должен их обнаруживать и добавлять в раскрывающийся список. (по умолчанию файлы хранятся в Documents/SoLyP/langs/  или аналогично)
- Ручной BPM (если хост не предоставляет синхронизацию, по умолчанию галочка снята, а поле ввода BPM не активно).
- MIDI-канал входа (1–16, Omni по умолчанию), но вообще может быть выбрано на дорожке хоста, где лежит плагин.
- Опция «Всегда наверху» (окно поверх других, по умолчанию включено), но при наведении на окно должна быть возможность его свернуть на панель задач и вернуть обратно. Важно понимать, что из-за настроек DAW есть вероятность, что при переходе на другую дорожку, окно плагина может быть свёрнуто, это недопустимый момент. Особенно при живом выступлении. Поэтому окно должно уметь быть "поверх всех окон" в любом случае. В самом крайнем случае придётся создать автономный модуль-приложение для окна лирики, и через API показывать на нём текст. И этот модуль уже можно будет и свернуть и развернуть и закрепить поверх всех окон.

### 3.7 UI / UX требования

- Основное окно: крупный текст, фоновый цвет – тёмный (настраиваемо), текст – светлый (настраиваемо).
- Возможность настройки разного цвета текста под разные секции (по умолчанию выключено, но если включено, то настройка будет в виде парных полей, например (| припев | #FCFCFC |), при этом указанный текст, должен выступать в качестве маски регулярного выражения, которое понимает, что маска "припев", может быть и секцией "[припев 1]" и "[припев 2]". Вместе с RGB надписью цвета должен быть и индикатор (стоящий рядом квадратик) указанного цвета, нажав на который можно выбрать любой другой цвет.
- При наведении мыши в правый верхний угол – появляются ползунки: количество видимых строк (2-10), размер шрифта (20-200 pt).
- Отдельная кнопка/иконка для открытия редактора текста и настроек.
- В режиме паузы – отображение кастомной надписи и предстрок.
- При ошибке (невалидный JSON, отсутствие секции или нарушение разметки) – красное сообщение на экране.

### 3.8 Поддержка Standalone режима

- Получает MIDI с любого выбранного устройства (MIDI-интерфейс).
- Синхронизация тактов – по MIDI Clock (Start/Continue, Clock импульсы). Плагин вычисляет такты (24 PPQN → деление на 24, сброс по Start).
- При отсутствии MIDI Clock – может работать по ручному BPM с кнопкой Tap Tempo.

## 4. Технические ограничения и требования

- Код на C++ (JUCE).
- Минимальные задержки обработки MIDI (блоки 64–128 сэмплов).
- Не обрабатывает аудио (только MIDI).
- Память: загрузка нескольких JSON (локаль, настройки, песня (размер до 1 МБ)).
- Все файлы локалей, настроек и песен (в подпапке songs) хранятся в пользовательской директории (Documents/SoLyP или аналогично).
- При сборке Standalone – возможность иконки в трее/доке и меню по ПКМ.

## 5. Этапы разработки (рекомендуемые)

1. Прототип: базовый VST3, окно с текстом, ручное переключение строк по MIDI-нотам, синхронизация по тактам, локализация
2. Добавление JSON-парсера, легенды, ориентиров/триггеров.
3. Реализация паузы, предстрок, гибридного режима.
4. UI-настройки, редактор текста (простой парсер секций).
5. Поддержка MIDI Clock в Standalone.
6. Полировка, тестирование на Ableton Live и других DAW.

## 6. Тестирование

- Проверить синхронизацию при изменении BPM.
- Переключение секций во время воспроизведения.
- Поведение при паузе и последующем Play.
- Длинные строки с кириллицей.
- Загрузка повреждённого JSON – сообщение об ошибке.
- Работа в Session View Ableton Live.
